# OpenCV with contrib modules — used for camera capture, frame
# preprocessing, and the JPEG/MJPEG render path. We previously also

# the upstream AprilTag algorithm and is meaningfully worse at
# distance: it loses tags 5-10 ft sooner than WPILib's wrapper around
# the C++ AprilTag library does. We now prefer robotpy-apriltag
# (same lib PhotonVision uses) and fall back to cv2.aruco only if it
# can't be imported.

# pyapriltags: a maintained fork of pupil-apriltags / dt-apriltags
# that publishes aarch64 wheels for cp310/cp311/cp312. Same upstream
# C++ AprilTag library that PhotonVision and WPILib use, so detection
# behavior matches what FRC teams running PhotonVision are getting.
#
# Why this dance: we tried robotpy-apriltag first (manylinux_2_35
# wheels, Bookworm+ only), then pupil-apriltags (no aarch64 wheels at
# all). pyapriltags is the only Python wrapper around the upstream
# AprilTag lib that publishes aarch64 wheels for current Python
# versions. If pyapriltags ever stops shipping wheels too, the
# manual-install + bridge paths now also include cmake/gcc so pip
# can build from sdist (slow but unblocks a fresh Pi).
